mmacommon.util.image module¶

extract_contours(image, tolerance=2.0)[source]¶

Return contour polygons for all objects within the image.

The methods extracts closed polygons for the contours of all white blobs within the binary input image and smoothens the contour polygons using the Douglas-Peucker algorithm. References:

  • https://en.wikipedia.org/wiki/Ramer%E2%80%93Douglas%E2%80%93Peucker_algorithm
  • http://scikit-image.org/docs/dev/api/skimage.measure.html#skimage.measure.find_contours
  • http://scikit-image.org/docs/dev/auto_examples/edges/plot_polygon.html#example-edges-plot-polygon-py
Parameters:
  • image (numpy.array(dtype=int)) – Binary image with 0..n objects.
  • tolerance (float) – Smoothing of the polygon. 0 means no smoothing.
Returns:

Closed polygons around contours of each image object.
Return type:

Generator over (n,2)-ndarrays.

Each contour is an ndarray of shape (n, 2), consisting of n (column, row) coordinates along the contour.

parse_data_url(data_url)[source]¶

Parses a data URL and returns its components.

Data URLs are defined as follows:

dataurl    := "data:" [ mediatype ] [ ";base64" ] "," data
mediatype  := [ type "/" subtype ] *( ";" parameter )
data       := *urlchar
parameter  := attribute "=" value

This specific implementation is limited to data urls of the following format:

dataurl := "data:" type "/" subtype ";base64" "," data
Here an example::
‘data:image/jpg;base64,/9j/4SXNRXhpZg...AUAFFABRQB//Z‘

References:

  • http://tools.ietf.org/html/rfc2397
  • http://www.askapache.com/online-tools/base64-image-converter/
Parameters:data_url (str) – A data URL.
Returns:Components of the data URL, e.g. (‘image’, ‘png’, ‘base64’, ‘/9j/4S’)
Return type:tuple (datatype, subtype, encoding, data)
Raises:ValueError if data URL doesn’t follow spec.

mmacommon.util.orbitio module¶

class OrbitIO(in_data, logger=None)[source]¶

Bases: object

Encapsulates parsing and construction of JSON/dictionary data structures that are used for communication between Orbit workers. It ensures that input and output data match the schema definition in registration.json and simplifies data access through data paths, e.g.

>>> in_data = {'data': {'image':'data:image,/9j/4SXNRFABRQB//Z'}}
>>> io = OrbitIO(in_data)
>>> io.read('data/image')
'data:image,/9j/4SXNRFABRQB//Z'

>>> io.write('result/data/grade', 42)
>>> io.write('result/data/label', 'grade')
>>> io.response()
[{'result': {'data': {'grade': 42, 'label': 'grade'}}}]

write_geo(kind, coordinates, **props)[source]¶

Writes a geo feature, e.g. polygon coordinates and properties.

Features are written in GeoJSON format. For details see http://geojson.org/geojson-spec.html. Here an example:

io.write_geo('Point', [1.0, 2.0], label='My point')
Parameters:
  • kind (str) – Type of geo feature. Must be one of ‘Point’, ‘LineString’, ‘Polygon’, ‘MultiPoint’, ‘MultiLineString’, ‘MultiPolygon’
  • coordinates (list) – Coordinates for this feature. See: http://geojson.org/geojson-spec.html
  • props (kwargs) – Arbitrary feature properties.
